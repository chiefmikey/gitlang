# GitLang Context — 2026-06-26

## Current State

Version 5.0.0 base, hardened since. Production build succeeds; ESLint passes clean
(0 errors, warnings only). Full unit (Vitest) and E2E (Playwright) coverage in place.
All GitHub issues from the v5.0.0 modernization closed.

**Production architecture:** S3 + CloudFront (client) + Lambda Function URL (API), all
behind a single CloudFront distribution at gitlang.net. No EC2, no separate API domain.
CloudFront routes `/gitlang/github/*` -> Lambda, `*` -> S3.

**Auth:** GitHub App (GitLang Stats, App ID 2992472) issuing installation access tokens
via `@octokit/auth-app`. Credentials resolve from env vars (`GH_APP_ID`,
`GH_INSTALLATION_ID`, `GH_PRIVATE_KEY`) first, then AWS Secrets Manager (`AUTH_GITLANG`,
region `us-east-2`). Tokens are cached in the Lambda warm container and refreshed ~5min
before expiry. (Note: the old comma-separated `GH_PAT` round-robin from early v5 is gone —
auth is App-installation tokens only.)

## Recent Changes (since v5.0.0)

### Production auth hardening (2026-06-25)
- **Concurrency guards** in `auth.ts`: in-flight promise dedup so concurrent requests at
  token expiry trigger only one token generation and one Secrets Manager fetch
  (thundering-herd / rotation-race prevention).
- **Bad-expiry guard:** a missing/unparseable `expiresAt` previously produced `NaN`, which
  made the token look permanently expired and silently regenerate on every call. Now it
  logs a structured `bad_expiry` error and falls back to a conservative 10-minute TTL.
- **Structured JSON logging** distinguishing `credentials_missing`, `aws_fetch_error`,
  `token_generation_error`, and `bad_expiry`, so auth failures are diagnosable from
  CloudWatch. `lambda.ts` adds structured `handler_error` logging with request path +
  operation tag (`logHandlerError` helper).
- Covered by `tests/server/helpers/github/auth.hardening.test.ts` (dedup, NaN-expiry guard,
  each structured-failure path).

### Test coverage
- Full unit + E2E suites added (`tests/client/`, `tests/server/`, `tests/e2e/`).
- **Gotcha:** mocks instantiated with `new` (e.g. `SecretsManagerClient`) must be `class`
  mocks, not `vi.fn(function(){})` — lint autofix rewrites the latter into a
  non-constructable arrow. `tests/**` is intentionally outside the `npm run fix` lint scope.
  (Both recorded in CLAUDE.md.)

## Architecture Notes

### Input Syntax
- `user` — all repos for user
- `user/repo` — single repo
- `user/repo1,repo2` — multiple repos
- `user1+user2` (or space-separated) — compile (aggregate)
- `user1~user2` — compare (side-by-side)
- `user/repo@author` — contributor's commit-level language breakdown
- `@user` or `org:name` — explicit user/org prefix

### Key Files
- `server/lambda.ts` — AWS Lambda handler dispatching all API routes (production)
- `server/requests/gitlangRouter.ts` — Koa router (local dev), mirrors Lambda routes
- `server/helpers/github/auth.ts` — GitHub App auth via `@octokit/auth-app` + Secrets Manager,
  with concurrency guards + structured failure logging
- `server/helpers/github/contributors.ts` — commit-level language analysis via file-extension mapping
- `server/helpers/github/rateLimit.ts` — rate limit info endpoint
- `server/helpers/github/languages.ts` — single Octokit instance, parallel language fetching
- `server/helpers/github/repositories.ts` — user/org repo listing with `per_page: 100`
- `client/lib/data/index.ts` — data pipeline: parsing -> fetching -> aggregation -> breakdown
- `client/lib/data/requests/merged.ts` — combined repos + langs request (perf optimization)
- `infra/template.yaml` — SAM template for Lambda deployment

### Data Flow
1. Input parsed by `parseEntry()` -> `{owner, repos, author}`
2. Groups split by `~` (compare) or processed as a single group
3. Within groups, entries split by `+` (compile/aggregate)
4. Per entry:
   - Full-user lookups (no specific repos, no author): `/merged` endpoint (single call)
   - Specific repos: `/repos` then `/langs` (two calls)
   - Contributor mode (`@author`): `/repos` then `/contributor-langs` per repo
5. `buildLangRepoBreakdown()` preserves per-repo attribution for expandable details
6. Language bars are clickable to toggle the per-repo breakdown

### Performance Notes
- `/merged` (GraphQL-backed, cached 5min) eliminates a client->server round trip for the common case
- Single Octokit instance reused across parallel language fetches (no per-repo instantiation)
- `per_page: 100` reduces pagination API calls for users with many repos
- CSS `zoom` used instead of `transform: scale` for responsive scaling (cleaner subpixel rendering)
- Forks are always excluded
