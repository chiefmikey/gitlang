## Architecture Notes

### Input Syntax
- `user` — all repos for user
- `user/repo` — single repo
- `user/repo1,repo2` — multiple repos
- `user1 user2` or `user1+user2` — compile (aggregate)
- `user1 ? user2` — compare (side-by-side)
- `user/repo@author` — contributor's commit-level language breakdown
- `@user` or `org:name` — explicit user/org prefix

### Key Files
- `server/lambda.ts` — AWS Lambda handler dispatching all API routes (production)
- `server/requests/gitlangRouter.ts` — Koa router (local dev), mirrors Lambda routes
- `server/helpers/github/auth.ts` — GitHub App auth via @octokit/auth-app + AWS Secrets Manager
- `server/helpers/github/contributors.ts` — commit-level language analysis via file extension mapping
- `server/helpers/github/rateLimit.ts` — rate limit info endpoint
- `server/helpers/github/languages.ts` — single Octokit instance, parallel language fetching
- `server/helpers/github/repositories.ts` — user/org repo listing with per_page: 100
- `client/lib/data/index.ts` — data pipeline: parsing → fetching → aggregation → breakdown
- `client/lib/data/requests/merged.ts` — combined repos + langs request (performance optimization)
- `infra/template.yaml` — SAM template for Lambda deployment

### Data Flow
1. Input parsed by `parseEntry()` → `{owner, repos, author}`
2. Groups split by `~` (compare) or processed as single group
3. Within groups, entries split by `+` (compile/aggregate)
4. Per entry:
   - Full-user lookups (no specific repos, no author): uses `/merged` endpoint (single call)
   - Specific repos: uses `/repos` then `/langs` (two calls)
   - Contributor mode (@author): uses `/repos` then `/contributor-langs` per repo
5. `buildLangRepoBreakdown()` preserves per-repo attribution for expandable details
6. Language bars clickable anywhere to toggle per-repo breakdown

### Performance Notes
- `/merged` endpoint eliminates one client→server round trip for the common case
- Single Octokit instance reused across parallel language fetches (no per-repo instantiation)
- `per_page: 100` reduces pagination API calls for users with many repos
- CSS `zoom` used instead of `transform: scale` for responsive scaling (cleaner subpixel rendering)
